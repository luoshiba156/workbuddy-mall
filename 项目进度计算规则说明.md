# 项目进度计算规则说明

> 适用页面：`project-dashboard.html` 甘特图项目进度表
> 基准日期：2026-05-21（今日）

---

## 一、项目数据结构

每个项目包含以下字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| `name` | 项目名称 | 智慧园区建设项目 |
| `projectStart` | 计划开始日期 `{month, day}` | `{month:5, day:5}` |
| `projectEnd` | 计划结束日期 `{month, day}` | `{month:5, day:30}` |
| `actualStart` | 实际开始日期 `{month, day}` 或 `null` | `{month:5, day:7}` |
| `completed` | 是否已完成（`'early'`/`'onTime'`/`'delayed'`） | 无则进行中 |
| `completedDate` | 实际完成日期（已完成必有） | `{month:5, day:21}` |

## 二、甘特图单元格颜色规则

用于 `getDateStatus()` 函数，决定每个日期单元格显示的背景色。

### 2.1 进行中 / 未开始项目

```
优先级从高到低：

① 红色    →  项目结束日期 + 1   ～  今日
② 绿色    →  实际开始日期  ～  min(今日, 项目结束日期)
③ 黄色    →  项目结束日期 + 1  开始，持续  (实际开始 - 项目开始) 天
④ 灰色    →  项目开始日期  ～  项目结束日期
⑤ 无色    →  其余日期
```

### 2.2 已完成项目

```
优先级从高到低：

① 暗红色  →  项目结束日期 + 1  ～  实际完成日期
② 暗蓝色  →  实际开始日期  ～  min(实际完成日期, 项目结束日期)
③ 灰色    →  项目开始日期  ～  项目结束日期
④ 无色    →  其余日期
```

## 三、hover 提示状态计算规则

用于 `getProjectTooltip()` 函数，hover 时显示项目状态。

### 3.1 已完成项目

```
延期完成天数 = 实际完成日期 - 项目结束日期
提前完成天数 = 项目结束日期 - 实际完成日期
```

| 条件 | 状态文字 | 颜色圆点 |
|------|----------|----------|
| 延期完成天数 > 0 | 已完成，延期 X 天 | 红色 |
| 延期完成天数 = 0 | 已完成，按时交付 | 绿色 |
| 延期完成天数 < 0 | 已完成，提前 X 天 | 绿色 |

### 3.2 进行中项目

**核心公式：**

```
已延期天数 = max(0, 今日日期 - 项目结束日期)

可能延期天数 = max(0, (实际开始日期 - 项目开始日期) - 已延期天数)
```

> 理解：项目一开始就晚了几天，产生了一个"初始延迟"。"可能延期天数" = 初始延迟中还没被"已延期"消耗掉的部分。

**判断逻辑：**

```
① 可能延期 > 0 且 已延期 > 0  →  已延期 X 天，还可能延期 Y 天  （红色圆点）
② 可能延期 > 0 且 已延期 ≤ 0  →  可能延期 Y 天                （黄色圆点）
③ 可能延期 ≤ 0 且 已延期 > 0  →  已延期 X 天                   （红色圆点）
④ 两者都 = 0，有实际开始日期   →  进行中                      （绿色圆点）
⑤ 无实际开始日期              →  未开始                      （灰色圆点）
```

### 3.3 未开始项目

```
今日 < 项目开始日期 → 未开始
```

## 四、hover 提示展示内容

hover 到项目行时，tooltip 展示以下信息：

| 行 | 内容 | 示例 |
|----|------|------|
| 1 | 项目日期 | 2026-05-05 ～ 2026-05-30 |
| 2 | 开始日期（实际） | 2026-05-07 / 未开始 |
| 3 | 完成日期（仅已完成项目） | 2026-05-21 |
| 4 | 项目状态（带颜色圆点） | ● 可能延期 2 天 |
